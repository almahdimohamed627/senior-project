Post-Verifier API Documentation

Endpoints

POST /verify (API Port)

Verifies a photo based on EXIF metadata.

  • Method: POST
  • Content-Type: multipart/form-data
  • Parameters:
    • image: File (JPEG/PNG)
    • doctorDevice: JSON string, e.g., {"model": "iPhone 12"}
  • Response: JSON {status: "accepted"|"not accepted"|"unstable", reasons: [...]}

GET /dashboard (Dashboard Port)

Access the preferences dashboard (requires auth).

GET /config (Dashboard Port)

Get API port config (requires auth).

Response: JSON {apiPort: number}

GET /docs (API Port)

Access API documentation.

GET /api/prefs (API Port)

Get current preferences.

POST /api/prefs (API Port)

Update preferences.

Body: JSON with check states and maxPhotoAgeMonths.

Check States

  • disabled: Check ignored.
  • not_necessary: If fails, status="unstable".
  • necessary: If fails, status="not accepted".

Statuses

  • accepted: All necessary checks pass.
  • not accepted: One or more necessary checks fail.
  • unstable: Non-critical issues; admin review needed.
